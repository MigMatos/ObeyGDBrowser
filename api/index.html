API Overview

Welcome to the API documentation. This guide provides an overview of available endpoints, authentication methods, and data formats.

Authentication

To authenticate, include the following headers in your requests:

  • OGDW_APPTOKEN: Your application token.
  • OGDWAPPNAME: Your application name.

Endpoints

Roles API

Endpoint: /roles

Methods: GET, POST

Manage user roles.

Parameters:

  • role_id (optional, int): The ID of the role to fetch or modify.
  • name (POST only, string): The name of the role to create.

Search API

Endpoint: /search

Methods: GET

Search for items in the database.

Parameters:

  • query (required, string): The search term.
  • limit (optional, int): Number of results to return.

Songs API

Endpoint: /songs

Methods: GET, POST, DELETE

Manage songs in the database.

Parameters:

  • song_id (optional, int): The ID of the song to fetch, update, or delete.
  • title (POST only, string): The title of the song to add.
  • artist (POST only, string): The artist of the song to add.

Tokens API

Endpoint: /tokenApps

Methods: POST

Manage application tokens.

Parameters:

  • app_name (required, string): The name of the application.
  • app_description (optional, string): Description of the application.
